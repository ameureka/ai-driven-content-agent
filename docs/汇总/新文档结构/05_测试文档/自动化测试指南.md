### 3. Playwright端到端测试

**文件**: `workflow-selector.spec.js`

**测试场景**:
- 工作流选择器渲染测试
- 工作流切换功能测试
- 用户交互流程测试

**核心测试用例**:
```javascript
test('应该正确渲染工作流选择器', async ({ page }) => {
    // 检查工作流选择器容器
    const workflowSelector = page.locator('.workflow-selector-section');
    await expect(workflowSelector).toBeVisible();
    
    // 检查默认工作流卡片
    const urlWorkflowCard = page.locator('[data-workflow="dify-general"]');
    await expect(urlWorkflowCard).toBeVisible();
});

test('应该能够选择不同的工作流', async ({ page }) => {
    // 测试工作流切换功能
    const textWorkflowCard = page.locator('[data-workflow="dify-article"]');
    await textWorkflowCard.click();
    await expect(textWorkflowCard).toHaveClass(/active/);
});
```

**运行方法**:
```bash
# 安装Playwright
npm install @playwright/test

# 运行测试
npx playwright test workflow-selector.spec.js

# 运行测试并显示浏览器
npx playwright test workflow-selector.spec.js --headed
```

## 🚀 持续集成配置

### GitHub Actions工作流

**文件**: `.github/workflows/test.yml`
```yaml
name: 自动化测试

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    timeout-minutes: 60
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: lts/*
    - name: 安装依赖
      run: npm ci
    - name: 安装Playwright浏览器
      run: npx playwright install --with-deps
    - name: 运行单元测试
      run: npm run test:unit
    - name: 启动服务器
      run: npm run dev &
    - name: 等待服务器启动
      run: npx wait-on http://localhost:8787
    - name: 运行Playwright测试
      run: npx playwright test
    - uses: actions/upload-artifact@v4
      if: always()
      with:
        name: playwright-report
        path: playwright-report/
        retention-days: 30
```

## 📚 最佳实践

### 测试编写原则

1. **独立性**: 每个测试应该独立运行
2. **可重复性**: 测试结果应该一致
3. **清晰性**: 测试意图应该明确
4. **快速反馈**: 单元测试应该快速执行
5. **真实性**: 测试场景应该接近真实使用

### 测试数据管理

// 使用工厂函数创建测试数据
