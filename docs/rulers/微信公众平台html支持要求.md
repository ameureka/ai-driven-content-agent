一、图片规范
尺寸上限：1080px * 4096px。超过这个尺寸的图片会被微信压缩，可能导致模糊。
大小上限：10M。超过这个大小的文件将无法上传。
推荐格式：PNG 24位。通常比 JPG 格式更清晰，尤其适合包含文字和色块的图片。
色彩模式：RGB 24位。必须使用网页和屏幕的色彩模式，如果使用印刷用的 CMYK 模式，颜色会变暗、失真。
长图建议：宽度建议为 900px (最大不超过1080px)，高度建议在 4500px 以内，以保证最佳清晰度。
二、文字与排版规范
小标题 (H2/H3)：
字号：建议 16-18号。
字数：建议一行不超过14个汉字，一般控制在8个汉字以内最佳。
正文 (P)：
字号：建议 14-16号。阅读最舒适的范围。想更精致可以用12号，但不建议更小。
行长：建议一行约20-24个汉字。
字间距：
建议值：1-2。设置合适的字间距会让排版显得更高级、不拥挤。
两端缩进：
建议值：1-2个字符的距离。常用于配合图片进行调整，让图文混排更精致。

处理的核心思路
CSS “炼金术”：将您所有基于类(class)和标签选择器的精美 CSS，全部提炼成行内样式 (inline style)，注入到每一个 HTML 标签中。这是适配微信的唯一可靠途径。
JS “静态化”：移除所有客户端 JavaScript (<script>)。动态导航这种功能，我们将用静态生成的方式来模拟，在渲染时直接生成一个静态的目录列表。
动态标签“占位符化”：您创造的 !video[...] 标记非常棒！我们将它和图片标记一起，改造为我们在微信中可以轻松识别和替换的**“占位符 <div>”**。
字体“本土化”：移除对 Google Fonts 的外部依赖 (<link>)，改用在所有手机上都表现良好的通用字体栈。

好的，这是一个非常重要且实际的问题。微信公众平台的文章编辑器**不是一个标准的网页浏览器环境**，它是一个高度管控和定制化的内容发布系统。因此，直接在其中使用通用的 JS 和 HTML 代码会受到**极其严格的限制**。

您必须将它理解为一个“富文本编辑器”，而不是一个“网页开发IDE”。

以下是您需要注意的核心事项和限制要求：

---

### **核心原则：安全第一，体验可控**

微信的首要目标是保护用户安全和维持平台的体验一致性。因此，它会无情地“清洗”或“过滤”掉任何可能带来安全风险或破坏排版样式的代码。

### **HTML 限制**

您可以将 HTML 看作是基础的“骨架”，但很多“高级器官”会被移除。

#### **✔️ 通常支持的标签：**

*   **基础文本格式化**：`<p>`, `<h1>` - `<h6>`, `<b>`, `<strong>`, `<i>`, `<em>`, `<u>`, `<br>`, `<hr>`
*   **列表**：`<ul>`, `<ol>`, `<li>`
*   **引用**：`<blockquote>`
*   **图片**：`<img>` (但 `src` 可能会被微信服务器转存，所以不要依赖原始路径)
*   **链接**：`<a>` (但微信会对链接进行安全检查，且在 App 内的打开方式受微信控制)
*   **表格**：`<table>`, `<tr>`, `<td>`, `<thead>`, `<tbody>` (基本支持，但复杂样式可能失效)

#### **❌ 绝对会被过滤掉的标签 (非常重要)：**

*   **`<script>`**：**这是第一天条，任何 `<script>` 标签都会被直接移除，无论内外链。** 这意味着您无法在文章中直接运行任何自定义 JavaScript。
*   **`<iframe>`**：绝对禁止，防止嵌入不安全的外部网页。
*   **`<form>`**：表单相关的标签如 `<form>`, `<input>`, `<textarea>`, `<button>` 都会被过滤，无法在文章内收集用户信息。
*   **`<link>` 和 `<meta>`**：无法引入外部 CSS 文件或定义页面元信息。
*   **`<object>`, `<embed>`, `<applet>`**：所有用于嵌入外部应用或多媒体的标签都会被移除。

---

### **JavaScript (JS) 限制**

这是最严格的部分。可以简单概括为：**在文章编辑器中，您完全无法使用自定义 JavaScript。**

*   **行内 JS 失效**：所有 `onclick`, `onload`, `onerror` 等事件处理属性都会被移除。例如 `<div onclick="alert('hello')"></div>` 会变成 `<div></div>`。
*   **`javascript:` 伪协议失效**：类似 `<a href="javascript:alert('hello')">` 的链接会被视为无效或不安全链接。
*   **无法调用任何 Web API**：如 `fetch`, `XMLHttpRequest`, `localStorage`, `setTimeout` 等等，因为根本没有 JS 的运行环境。

---

### **CSS 限制**

CSS 的情况比 JS 稍好，但同样有诸多限制。

#### **✔️ 通常支持的方式：**

*   **行内样式 (Inline Styles)**：这是最可靠、最受推荐的方式。例如 `<p style="color: red; font-size: 16px;">这是红色的文字</p>`。

#### **❌/⚠️ 受限或不支持的方式：**

*   **`<style>` 标签**：虽然有时某些简单的 `<style>` 规则能够生效，但非常不可靠。微信随时可能过滤掉整个标签，或者移除其中的它认为有风险的规则（如 `position: fixed` 等可能破坏页面布局的样式）。**强烈不建议依赖此方式**。
*   **外部样式表**：如前所述，`<link>` 标签会被移除，无法加载外部 CSS 文件。
*   **复杂选择器和伪类**：过于复杂的 CSS 选择器、动画 (`@keyframes`)、以及部分伪类（如 `:hover`）在移动端的支持可能不佳或被过滤。
*   **单位**：建议使用 `px` 和 `%`。像 `rem`, `em`, `vw`, `vh` 等现代 CSS 单位的支持性可能不一致。

---

### **总结与正确实践**

1.  **思维转变**：不要把微信文章当作一个网页来制作。把它当作一份“可以用 HTML 来排版的 Word 文档”。
2.  **静态内容为主**：您的内容应该是纯静态的图文展示。
3.  **样式靠行内**：所有样式都应该尽可能通过 `style="..."` 属性写在 HTML 标签上。
4.  **禁止任何交互**：不要试图在文章本身添加点击事件、表单等交互功能。

### **“如果我确实需要交互功能，怎么办？”**

这是问题的关键。微信把这条路堵死的同时，也开了一扇窗：

*   **原文链接/公众号菜单**：将用户引导到您**自己的、已备案的、配置好 JS-SDK 的网页**上。在这个网页里，您可以拥有几乎所有的 HTML, CSS 和 JS 控制权。
*   **微信 JS-SDK**：在您自己的网页上，可以通过引入微信的 JS-SDK，来调用微信的原生能力，如分享、支付、扫一扫、获取地理位置等。
*   **小程序**：对于复杂的交互和功能，最佳的载体是小程序。您可以在文章中嵌入小程序卡片，引导用户跳转。

**简而言之**：微信文章本身是“只读”的静态展示区，所有“可写”的动态交互都必须在外部网页或小程序中完成。
