# 前后端开发与测试总结

## 项目概述

本项目是一个AI驱动的内容生成代理，包含前端界面和后端API服务，集成了Dify工作流来生成文章内容。

## 前后端设计架构

### 后端API设计

#### 1. API路由结构
- **主要端点**: `/api/v1/workflows/dify-article/execute`
- **认证方式**: 使用`X-API-Key`头进行API密钥验证
- **请求格式**: POST请求，JSON格式数据
- **响应格式**: 流式响应(Server-Sent Events)

#### 2. 核心组件
- **routes.js**: 主要路由处理器
  - `validateApiKey()`: API密钥验证函数
  - `executeWorkflow()`: 工作流执行函数
  - 错误处理和响应格式化
- **difyArticle.js**: Dify工作流集成
- **templateManager.js**: 模板管理服务

#### 3. 安全设计
- API密钥验证机制
- CORS配置支持跨域请求
- 错误信息安全处理

### 前端设计

#### 1. 文件结构
- **index.html**: 主界面
- **script.js**: 核心JavaScript逻辑
- **styles.css**: 样式文件
- **wiki.html**: 文档页面

#### 2. 前端调用逻辑
```javascript
// API调用流程
1. 用户输入API密钥和文章标题
2. 构建请求URL: `${protocol}//${host}/api/v1/workflows/dify-article/execute`
3. 设置请求头: 'X-API-Key': apiKey
4. 发送POST请求包含title、style、context
5. 处理流式响应数据
6. 重试机制: 最多3次重试
7. 失败时使用模拟数据
```

#### 3. 关键函数
- **fetchWithRetryPost()**: 带重试的POST请求函数
- **generateArticleContent()**: 文章生成主函数
- **handleStreamResponse()**: 流式响应处理

## 开发过程中的问题与解决方案

### 1. API密钥验证问题

**问题**: 前端发送请求但后端没有显示API密钥验证的调试信息

**排查过程**:
1. 检查前端是否正确设置`X-API-Key`头 ✓
2. 检查后端路由是否正确配置 ✓
3. 添加调试日志到`validateApiKey`函数
4. 添加调试日志到`executeWorkflow`函数入口

**解决方案**: 
- 在关键函数添加详细的调试日志
- 确保服务器重启后代码更改生效

### 2. 前后端连接问题

**问题**: 前端显示403、503、ERR_CONNECTION_REFUSED等错误

**排查方法**:
1. 使用curl命令测试API端点
2. 检查服务器是否正常运行
3. 验证CORS配置
4. 检查端口占用情况

### 3. 缓存问题

**问题**: 浏览器缓存导致代码更改不生效

**解决方案**:
- 使用`window.location.reload(true)`强制刷新
- 清除浏览器缓存
- 使用开发者工具的"Disable cache"选项

## 使用Playwright进行测试的经验

### 1. 可用的Playwright工具

虽然在本次调试中Playwright MCP工具不可用，但通常可以使用以下功能：
- **页面操作**: 点击、填充表单、导航
- **元素选择**: CSS选择器、XPath
- **状态检查**: 页面加载、元素可见性
- **日志获取**: 控制台日志、网络请求

### 2. 替代测试方法

当Playwright不可用时的替代方案：
1. **手动浏览器测试**: 直接在浏览器中操作
2. **curl命令测试**: 测试API端点
3. **JavaScript控制台**: 直接执行JavaScript代码
4. **服务器日志**: 监控后端日志输出

### 3. 测试流程设计

```javascript
// 理想的测试流程
1. 启动服务器
2. 打开浏览器页面
3. 填充API密钥
4. 填充文章标题
5. 点击生成按钮
6. 检查响应结果
7. 验证错误处理
8. 检查控制台日志
```

### 4. 常见测试错误与解决

#### 缓存问题
- **现象**: 代码更改后测试结果不变
- **解决**: 强制刷新页面，清除缓存

#### 网络连接问题
- **现象**: ERR_CONNECTION_REFUSED, 503错误
- **解决**: 检查服务器状态，验证端口配置

#### API认证问题
- **现象**: 403 Forbidden错误
- **解决**: 验证API密钥设置，检查请求头
