## 工作流状态判断机制

### 1. 工作流运行状态判断

**后端状态管理:**
- 通过`TransformStream`管理流式连接状态
- 使用`AbortController`支持取消操作
- 通过事件回调机制跟踪执行进度

**前端状态管理:**
```javascript
// 状态标志
let isGeneratingArticle = false;

// 状态指示器更新
workflowStatusIndicator.innerHTML = `<span class="status-dot active"></span> ${status}`;

// 按钮状态控制
generateArticleBtn.disabled = isGeneratingArticle;
```

### 2. 工作流完成判断

**完成条件:**
1. 接收到`workflow_finished`事件 (Dify API)
2. 接收到`type=complete`事件 (前端)
3. 接收到`[DONE]`信号
4. 接收到`done=true`状态
5. 流连接关闭

**完成处理:**
```javascript
function completeArticleGeneration() {
  isGeneratingArticle = false;
  generateArticleBtn.disabled = false;
  generateArticleBtn.innerHTML = '<i class="icon ion-md-create"></i> AI生成';
  workflowStatusIndicator.innerHTML = '<span class="status-dot completed"></span> 生成完成';
  
  // 5秒后隐藏状态指示器
  setTimeout(() => {
    if (!isGeneratingArticle) {
      workflowStatusIndicator.classList.add('hidden');
    }
  }, 5000);
}
```

### 1. 多层级结果提取

```javascript
const extractContent = (data) => {
  // 1. 优先从outputs提取
  if (data.outputs) return data.outputs;
  
  // 2. 从data对象的各个字段尝试提取
  const fields = ['text', 'content', 'result', 'answer', 'output'];
  for (const field of fields) {
    if (data[field]) return data[field];
  }
  
  return null;
};
```
