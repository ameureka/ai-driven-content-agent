# AI驱动内容代理系统 - 开发环境搭建指南

## 🚀 快速开始

### 系统要求

#### 基础环境
- **Node.js**: >= 18.0.0 (推荐使用 LTS 版本)
- **npm**: >= 8.0.0 或 **yarn**: >= 1.22.0
- **Git**: >= 2.30.0
- **操作系统**: macOS, Linux, Windows 10+

#### 推荐工具
- **IDE**: Visual Studio Code (推荐插件见下文)
- **终端**: iTerm2 (macOS) / Windows Terminal (Windows)
- **浏览器**: Chrome/Firefox (用于调试)
- **API测试**: Postman 或 Insomnia

### 环境检查脚本

```bash
#!/bin/bash
# check-environment.sh

echo "🔍 检查开发环境..."

# 检查 Node.js
if command -v node &> /dev/null; then
    NODE_VERSION=$(node --version)
    echo "✅ Node.js: $NODE_VERSION"
    
    # 检查版本是否满足要求
    REQUIRED_VERSION="18.0.0"
    if [ "$(printf '%s\n' "$REQUIRED_VERSION" "$NODE_VERSION" | sort -V | head -n1)" = "$REQUIRED_VERSION" ]; then
        echo "   版本满足要求 (>= $REQUIRED_VERSION)"
    else
        echo "   ⚠️  版本过低，请升级到 $REQUIRED_VERSION 或更高版本"
    fi
else
    echo "❌ Node.js 未安装"
    echo "   请访问 https://nodejs.org 下载安装"
fi

# 检查 npm
if command -v npm &> /dev/null; then
    NPM_VERSION=$(npm --version)
    echo "✅ npm: $NPM_VERSION"
else
    echo "❌ npm 未安装"
fi

# 检查 Git
if command -v git &> /dev/null; then
    GIT_VERSION=$(git --version)
    echo "✅ Git: $GIT_VERSION"
else
    echo "❌ Git 未安装"
    echo "   请访问 https://git-scm.com 下载安装"
fi

# 检查 Cloudflare CLI (可选)
if command -v wrangler &> /dev/null; then
    WRANGLER_VERSION=$(wrangler --version)
    echo "✅ Wrangler: $WRANGLER_VERSION"
else
    echo "⚠️  Wrangler 未安装 (部署到 Cloudflare 时需要)"
    echo "   运行: npm install -g wrangler"
fi

echo ""
echo "🎯 环境检查完成！"
```

## 📦 项目初始化

### 1. 克隆项目

```bash
# 克隆仓库
git clone https://github.com/your-org/ai-driven-content-agent.git
cd ai-driven-content-agent

# 检查分支
git branch -a
git checkout main
```

### 2. 安装依赖

```bash
# 使用 npm
npm install

yarn install

# 验证安装
npm list --depth=0
```

### 3. 环境配置

#### 创建环境变量文件

```bash
# 复制环境变量模板
cp .env.example .env.local

# 编辑环境变量
vim .env.local
```

#### 环境变量说明

```bash

# ===================
```

### Git 配置

#### Git Hooks 设置

```bash
# 安装 husky
npm install --save-dev husky

# 添加 commit-msg hook
```


#### Commit 规范

```bash
# .gitmessage 模板
# <type>(<scope>): <subject>
#
# <body>
#
# <footer>
```

```javascript
import { describe, it, expect, beforeEach } from 'vitest';
import { createTestServer } from '@tests/utils/test-server';
import { mockDifyAPI } from '@tests/mocks/dify';

describe('Content API', () => {

    mockDifyAPI();
  });

        content: 'Test content',
        workflow: 'dify-general'

        css: expect.any(String)
      }
    });
  });
});
```

## 🚀 部署准备

### Cloudflare Workers 配置

```toml
# wrangler.toml
name = "ai-content-agent"
main = "src/worker.js"
compatibility_date = "2024-01-01"
compatibility_flags = ["nodejs_compat"]

[env.development]
name = "ai-content-agent-dev"
vars = { NODE_ENV = "development" }

[env.staging]
name = "ai-content-agent-staging"
vars = { NODE_ENV = "staging" }

[env.production]
name = "ai-content-agent-prod"
vars = { NODE_ENV = "production" }

# KV 命名空间
[[kv_namespaces]]
binding = "CACHE"
preview_id = "your-preview-id"
id = "your-production-id"

# D1 数据库
[[d1_databases]]
binding = "DB"
database_name = "ai-content-agent"
database_id = "your-database-id"

# 环境变量
[vars]
API_VERSION = "v1"
MAX_REQUEST_SIZE = "10MB"
CACHE_TTL = "3600"
```

### 部署脚本

```bash
#!/bin/bash
# scripts/deploy.sh

set -e

ENV=${1:-staging}

echo "🚀 部署到 $ENV 环境..."

# 1. 运行测试
echo "📋 运行测试..."
npm run test

# 2. 构建项目
echo "🔨 构建项目..."
npm run build

# 3. 部署到 Cloudflare
echo "☁️ 部署到 Cloudflare Workers..."
wrangler publish --env $ENV

# 4. 运行部署后测试
echo "🧪 运行部署后测试..."
npm run test:e2e -- --env $ENV

echo "✅ 部署完成！"
```
