# AI驱动内容代理系统 - 测试策略文档

## 概述

本文档详细描述了AI驱动内容代理系统的测试策略、测试框架、测试用例设计和质量保证流程，确保系统的可靠性、稳定性和性能。

## 测试策略

### 测试金字塔

```
        E2E Tests (10%)
      ┌─────────────────┐
     │  端到端测试      │
    ┌───────────────────┐
   │  Integration Tests │ (20%)
   │    集成测试        │
  ┌─────────────────────┐
 │   Unit Tests (70%)   │
 │     单元测试         │
 └─────────────────────┘
```

### 测试分层策略

| 测试层级 | 覆盖范围 | 执行频率 | 工具 |
|----------|----------|----------|------|
| 单元测试 | 函数、类、组件 | 每次提交 | Vitest, Jest |
| 集成测试 | API、数据库、服务 | 每日构建 | Supertest, MSW |
| 端到端测试 | 完整用户流程 | 发布前 | Playwright |
| 性能测试 | 系统性能指标 | 每周 | K6, Lighthouse |
| 安全测试 | 安全漏洞扫描 | 每月 | OWASP ZAP |

```javascript
import { render, screen, fireEvent, waitFor } from '@testing-library/react';
import { vi } from 'vitest';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import ContentEditor from '../ContentEditor';
import { useContentStore } from '../../stores/contentStore';

// Mock store
vi.mock('../../stores/contentStore');

describe('ContentEditor', () => {
  let queryClient;
  const mockUseContentStore = vi.mocked(useContentStore);

  beforeEach(() => {
    queryClient = new QueryClient({
      defaultOptions: {
        queries: { retry: false },
        mutations: { retry: false }
      }
    });

    mockUseContentStore.mockReturnValue({
      content: '',
      setContent: vi.fn(),
      isLoading: false,
      error: null
    });
  });

  const renderWithProviders = (component) => {
    return render(
      <QueryClientProvider client={queryClient}>
        {component}
      </QueryClientProvider>
    );
  };

  it('should render content editor with initial state', () => {
    renderWithProviders(<ContentEditor />);
    
    expect(screen.getByRole('textbox')).toBeInTheDocument();
    expect(screen.getByText('重写内容')).toBeInTheDocument();
    expect(screen.getByText('生成文章')).toBeInTheDocument();
  });

  it('should update content when typing', async () => {
    const mockSetContent = vi.fn();
    mockUseContentStore.mockReturnValue({
      content: '',
      setContent: mockSetContent,
      isLoading: false,
      error: null
    });

    renderWithProviders(<ContentEditor />);
    
    const textarea = screen.getByRole('textbox');
    fireEvent.change(textarea, { target: { value: 'Test content' } });
    
    await waitFor(() => {
      expect(mockSetContent).toHaveBeenCalledWith('Test content');
    });
  });

  it('should handle rewrite action', async () => {
    const mockRewrite = vi.fn().mockResolvedValue({
      data: { rewrittenContent: 'Rewritten content' }
    });
    
    // Mock API call
    vi.mock('../../api/content', () => ({
      rewriteContent: mockRewrite
    }));

    mockUseContentStore.mockReturnValue({
      content: 'Original content',
      setContent: vi.fn(),
      isLoading: false,
      error: null
    });

    renderWithProviders(<ContentEditor />);
    
    const rewriteButton = screen.getByText('重写内容');
    fireEvent.click(rewriteButton);
    
    await waitFor(() => {
      expect(mockRewrite).toHaveBeenCalledWith({
        content: 'Original content',
        style: 'professional',
        tone: 'neutral'
      });
    });
  });

  it('should display error message when rewrite fails', async () => {
    mockUseContentStore.mockReturnValue({
      content: 'Test content',
      setContent: vi.fn(),
      isLoading: false,
      error: 'Rewrite failed'
    });

    renderWithProviders(<ContentEditor />);
    
    expect(screen.getByText('Rewrite failed')).toBeInTheDocument();
    expect(screen.getByRole('alert')).toHaveClass('error');
  });

  it('should show loading state during rewrite', () => {
    mockUseContentStore.mockReturnValue({
      content: 'Test content',
      setContent: vi.fn(),
      isLoading: true,
      error: null
    });

    renderWithProviders(<ContentEditor />);
    
    expect(screen.getByText('处理中...')).toBeInTheDocument();
    expect(screen.getByRole('button', { name: '重写内容' })).toBeDisabled();
  });
});
```

```javascript
          message: '服务器内部错误'

      });
    });
  });

  describe('POST /auth/register', () => {

        password: 'password123',
        name: 'New User'

      const createdUser = {
        id: '2',

        name: newUser.name
```

```javascript
const errorRate = new Rate('errors');

    errors: ['rate<0.01'],

  // 设置阶段：创建测试用户和获取认证令牌

  for (const user of testUsers) {
    const loginResponse = http.post(`${BASE_URL}/api/auth/login`, JSON.stringify(user), {
      headers: { 'Content-Type': 'application/json' },
    });
    
    if (loginResponse.status === 200) {
      const data = JSON.parse(loginResponse.body);
      tokens.push(data.data.token);
    }
  }
  
  return { tokens };
}

export default function(data) {
  // 随机选择用户令牌
  const token = data.tokens[Math.floor(Math.random() * data.tokens.length)];
  const headers = {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
```

```javascript
// performance/lighthouse-test.js
import lighthouse from 'lighthouse';
import * as chromeLauncher from 'chrome-launcher';
import fs from 'fs';

const urls = [
  'https://your-app.workers.dev/',
  'https://your-app.workers.dev/dashboard',
  'https://your-app.workers.dev/editor',
];

const config = {
  extends: 'lighthouse:default',
  settings: {
    onlyAudits: [
      'first-contentful-paint',
      'largest-contentful-paint',
      'first-meaningful-paint',
      'speed-index',
      'interactive',
      'cumulative-layout-shift',
      'total-blocking-time',
    ],
  },
};

async function runLighthouse() {
  const chrome = await chromeLauncher.launch({ chromeFlags: ['--headless'] });
  const options = {
    logLevel: 'info',
    output: 'json',
    onlyCategories: ['performance'],
    port: chrome.port,
  };
  
  const results = [];
  
  for (const url of urls) {
    console.log(`Running Lighthouse for ${url}`);
    
    const runnerResult = await lighthouse(url, options, config);
    const { lhr } = runnerResult;
    
    const metrics = {
      url,
      timestamp: new Date().toISOString(),
      performance: lhr.categories.performance.score * 100,
      metrics: {
        firstContentfulPaint: lhr.audits['first-contentful-paint'].numericValue,
        largestContentfulPaint: lhr.audits['largest-contentful-paint'].numericValue,
        firstMeaningfulPaint: lhr.audits['first-meaningful-paint'].numericValue,
        speedIndex: lhr.audits['speed-index'].numericValue,
        interactive: lhr.audits['interactive'].numericValue,
        cumulativeLayoutShift: lhr.audits['cumulative-layout-shift'].numericValue,
        totalBlockingTime: lhr.audits['total-blocking-time'].numericValue,
      },
    };
    
    results.push(metrics);
    
    // 检查性能阈值
    const thresholds = {
      firstContentfulPaint: 2000,
      largestContentfulPaint: 4000,
      speedIndex: 4000,
      interactive: 5000,
      cumulativeLayoutShift: 0.1,
      totalBlockingTime: 300,
    };
    
    console.log(`Performance Score: ${metrics.performance}`);
    
    for (const [metric, value] of Object.entries(metrics.metrics)) {
      const threshold = thresholds[metric];
      const status = value <= threshold ? '✅' : '❌';
      console.log(`${status} ${metric}: ${value} (threshold: ${threshold})`);
    }
  }
  
  await chrome.kill();
  
  // 保存结果
  fs.writeFileSync(
    'lighthouse-results.json',
    JSON.stringify(results, null, 2)
  );
  
  return results;
}

if (import.meta.url === `file://${process.argv[1]}`) {
  runLighthouse().catch(console.error);
}

export { runLighthouse };
```

## 测试最佳实践

### 1. 测试组织

- **按功能模块组织测试**：将相关的测试用例组织在一起
- **使用描述性的测试名称**：清楚地描述测试的目的和预期结果
- **遵循AAA模式**：Arrange（准备）、Act（执行）、Assert（断言）

### 2. 测试数据管理

- **使用测试工厂**：创建可重用的测试数据生成器
- **隔离测试数据**：每个测试使用独立的数据，避免相互影响
- **清理测试数据**：测试结束后清理创建的数据

### 3. Mock和Stub

- **合理使用Mock**：对外部依赖进行Mock，确保测试的独立性
- **保持Mock的真实性**：Mock的行为应该尽可能接近真实实现
- **避免过度Mock**：不要Mock被测试的核心逻辑

### 4. 测试覆盖率

- **追求有意义的覆盖率**：关注关键路径和边界条件
- **不盲目追求100%覆盖率**：重点关注业务逻辑的覆盖
- **定期审查覆盖率报告**：识别未测试的代码区域

### 5. 持续改进

- **定期重构测试代码**：保持测试代码的质量
- **监控测试执行时间**：优化慢速测试
- **分析测试失败原因**：持续改进测试的稳定性
